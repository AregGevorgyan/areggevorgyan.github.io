Fast computational geometry algorithms in WebAssembly.

This was an experiment to implement the "ear clipping" algorithm from computational geometry for polygon triangulation. Although easy to state, triangulation is surprisingly difficult, as concave polygons can be arbitrarily ill-behaved, and holes create a tree structure. This implementation works even for polygons with multiple holes and nested polygons, using some clever techniques due to David Eberly.

Although this O(N2) implementation is better than a naive O(N3) approach, it is not the best possible in theory, as there exists a randomized O(N log* N) algorithm (Seidel 1991). However, for most practical applications, ear clipping is fast enough and produces more robust results.

The algorithm was implemented in C++ from scratch, then compiled to WebAssembly using Emscripten to produce code that could be run in the browser.
